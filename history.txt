(2004/09/12) 發布新版 v3.0，這是第一個把 SSH / SFTP 整合到 BBMan 的版本 :) 新功能如下： 
請注意！！！SSH / SFTP 基本上還沒完全完工，但是基本功能都有了，理論上即使有問題應該也不至於造成 SSH server crash :p 
我用了很久是沒有什麼問題啦，只不過目前有極少部份 terminal protocol 不知道該如何處理，這意味著某些情況你的畫面會有極少部分有點小亂 :p 
簡單說明一下：SFTP 就是全程加密的 FTP，可對 SSH server 做上下傳檔案的動作，而且 SFTP 完全走 SSH protocol，不需要額外架設 FTP server ，僅需架設 ssh server 即可（一般 Linux / FreeBSD 電腦預設都會有安裝），相對於 SFTP，可能更有名氣的是 SCP ，這兩個基本上看成一樣的東西大概不會有什麼關係 :p

可以跟 BBS 我的最愛一樣，儲存密碼，而且密碼都會經過加密儲存，避免有人偷連你的 SSH :p 
SSH / SFTP 所有資料都經過加密（好像是廢話？！）和壓縮，事實上我拿 BBMan for windows 的 SFTP 傳輸速率跟 WinSCP 比較，發現傳輸速率 SFTP of BBMan 竟然比較快？！希望不是靈異現象 :p

(2004/08/05)
發布 v1.7.6，新增功能〔貼上時自動換行〕，若貼上的文字有某行長度超過所指定長度時，程式會自動換行
換行時會偵測中文字和英文字，使中文和英文不會被截掉一半，而且第一行會依據目前游標位置斷行，例如你游標已經在 x=70 的位置，那第一行如果超過 10 個字元就會換行，同時會自動忽略每一行開頭的空白
建議將長度指定為 76 ，因為目前 BBS 一行最大長度為 80 ，但每次被引言時會內縮兩個字元，一般回文時若無特殊指定只能引用前兩個人的文章，這時你的文章可能會內縮四個字元，80 - 4 = 76
請大家幫忙測試這個功能，因為這部份程式都是用指標指來指去的，若有地方寫錯很容易會 crash ，不過經過我測試多次結果目前並沒有任何問題就是了 XD
BBMan 開發到目前為止 BBS 相關的重要功能應該都已經算是完備了，目前還想做的只差標點符號輸入法而已，因此往後更新速度會比較慢（我好像常常這樣講，不過這次應該是真的 =.="），現在要多花點時間在課業上，不過如果有任何 bug 還是會盡快修復，至於對程式的建議如果我覺得有必要或是不麻煩的還是會採納的，總之我會繼續維護 BBMan 的，討論區我也都會去看 :D
關於 BBMan for FreeBSD 極少數字無法正常顯示的問題，這是因為若在 wxGTK for gtk2 欲顯示字型中沒有定義的字會 crash ，因此目前只能消極的把這些字忽略掉，可能會有人有疑問說為什麼 PCManX 都可以正常顯示不會 crash ，經過我測試過發現，PCManX 的方法只適用於 wxGTK + gtk1 而不適用於 wxGTK + gtk2 ，而目前 BBMan 是使用 wxGTK + gtk2 ，要移植到 wxGTK + gtk1 上有些問題，一方面是調整字型將是個問題（調整字型會停頓非常久），二來需要改寫一下 BBMan 才能使用選單快速鍵，再者 gtk2 比較新，介面比較漂亮，因此目前沒有打算移植到 gtk1 ，只能消極的等待 wxGTK 改善這問題了 ><" 至於 BBMan for Linux 則是沒有此一問題（我也很好奇為什麼 wxGTK + gtk2 在 Linux / FreeBSD 上會有不同的反應 =.="）
